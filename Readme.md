一、核心目标
开发极简的跳转界面，支持 “学号 + 密码” 登录，登录状态持久化（免重复登录），登录后可快速访问本地软件、外部平台及关联服务，无数据库依赖。
二、登录环节需求（新增持久化规则）
基础登录规则：
登录方式：学号 + 密码验证，预设 6 个固定测试账号（硬编码），每个账号对应 1-6 号虚拟机编号；
校验逻辑：账号密码匹配则登录成功并跳转应用入口页，不匹配则提示错误，未登录禁止访问应用入口页。
新增登录状态持久化：
登录成功后，浏览器自动保存登录状态（无需手动勾选 “记住我”），有效期可设为长期有效（如 30 天，或直到用户主动退出）；
下次打开浏览器访问该界面时，若检测到有效登录状态，直接跳过登录页，自动进入应用入口页；
状态存储要求：无需区分 Cookie / 本地存储，只需实现 “免登录” 效果，且状态与当前浏览器绑定（换浏览器 / 清空缓存需重新登录）。
三、应用入口页需求（补充退出逻辑）
页面展示：
顶部展示 “当前虚拟机编号 + 登录用户学号”，提供 “退出虚拟机” 按钮；
主体以卡片形式展示 8 个应用模块（IoT/IMS/VC/ 博图 / 数字化工厂 / Dify / 边缘服务器 / 智能教学 AI），点击触发对应跳转（本地软件 / 外部链接 / 关联服务 / 预留链接）。
模块跳转规则（8 个模块）：
模块名称	跳转类型	跳转目标
IoT 平台	外部链接	物联网设备管理与数据采集平台
IMS 平台	外部链接	智能制造执行系统
VC 软件	本地软件调用	本地 Windows10 电脑中的 VC 虚拟控制软件
博图软件	本地软件调用	本地 Windows10 电脑中的博图工业软件
数字化工厂	关联服务访问	Linux 服务器上的 WebGL 仿真系统
Dify	关联服务访问	Linux 服务器上的 AI 应用开发平台
边缘服务器	关联服务访问	Linux 服务器上的边缘网关管理界面
智能教学 AI	预留链接	待对接的智能教学辅助系统
退出功能：
点击 “退出虚拟机”，清除已保存的登录状态，并跳转回登录页；
退出后再次访问，需重新输入学号密码登录。
四、基础体验要求
跳转失败时给出简洁提示（如 “跳转失败，请检查配置”）；
界面布局简洁，模块卡片有基础交互反馈（如 hover 效果）；
适配校园网环境下 Windows10 系统的主流浏览器（Chrome/Edge 等）。
总结
核心新增需求：登录状态持久化，免重复登录，退出时清除状态；
核心约束：无数据库，仅通过浏览器本地存储实现状态保存；
关键逻辑：登录→保存状态→下次免登，退出→清除状态→需重新登录。
